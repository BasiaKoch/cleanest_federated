# Methodology — System Heterogeneity Experiment

> Drop-in methodology subsection for the thesis. Use as a sibling to the
> statistical-heterogeneity methodology in `thesis_ready/writing/02_methods.md`.

## Motivation

The proximal regulariser in FedProx was introduced primarily to handle
**γ-inexact local updates** — partial work submitted by clients that cannot
afford to perform the full $E$ local epochs per round
[Li et al., 2020, §3]. In real-world FL, clients have heterogeneous
hardware (mobile phones, hospital workstations, edge devices) and variable
network bandwidth, so the effective amount of local computation completed
each round differs across clients. The statistical-heterogeneity experiment
in §X.Y of this dissertation isolates the data-distribution axis; the
present section isolates the **computational-budget axis** of FL
heterogeneity.

## Experimental design

All hyperparameters and the data partition are held identical to the
headline statistical-heterogeneity sweep (`balanced_paired_7_clients`,
$R = 150$, $\mu = 0.01$, 10 paired seeds, batch size 32, SGD with lr 0.01
and momentum 0.9) — only the per-client per-round local-epoch budget
changes. This isolates the system-heterogeneity contribution.

Three conditions are compared:

**C0 — No system heterogeneity (baseline; already evaluated).** Every
client performs $E = 20$ local epochs every round. This is the headline
condition; numbers are reported in §X.Y. Inclusion here is for comparative
reference.

**C1 — Fixed stragglers.** Two designated clients (C5 and C6 in the
partition; see §X.X for partition composition) always perform $E_\text{straggler} = 5$
local epochs, while the other five clients perform $E = 20$. Straggler
identity is deterministic and constant across rounds and seeds. This setup
models a deployment where specific institutions have consistently weaker
compute (e.g., a small clinic with older hardware). This condition is
closer to Marija (2025) §3.8.4 but adapted to 7 clients.

**C2 — Random stragglers (Li et al. 2020 protocol).** Each communication
round, half of the participating clients (3 or 4 of 7) are randomly
designated as stragglers. Each straggler performs $E_i \sim \text{Uniform}[1, E_\max - 1]$
local epochs; non-stragglers perform $E_\max = 20$. Which clients are
stragglers in which rounds is sampled deterministically from a seeded
RNG, so paired FedAvg and FedProx runs see an identical straggler schedule
and the within-pair $\Delta$ reflects only the algorithmic difference. This
setup follows the canonical Li et al. (2020) §5.2 experimental design exactly.

## Implementation

The local-epoch schedule is generated by the function
`build_epoch_schedule` in `mnist_dermnist/fl/system_het.py`, which returns
a $(R \times K)$ integer array of per-(round, client) local epochs. The
schedule is seeded with `cfg.seed` so paired FedAvg and FedProx runs share
an identical schedule. The federated server loop
(`mnist_dermnist/fl/server_loop.py`) reads each entry $\text{schedule}[r, k]$
and passes it as the `num_epochs` argument to the per-client local
training function. When the configured mode is `"uniform"`, every entry
equals the global $E$ and behaviour is bit-identical to the
statistical-heterogeneity pipeline; the system-heterogeneity code path is
therefore a strict extension of the existing pipeline.

The proximal term and all other algorithmic logic are unchanged from the
headline experiments. FedProx still uses $\mu = 0.01$, selected on an
earlier discarded partition (not the partition or test set used in this
report), so the headline test comparison is protected against
hyperparameter selection bias.

## Metrics and statistical analysis

The primary outcome is test macro-F1 evaluated once per run at the round
of best validation macro-F1, identical to the statistical-heterogeneity
protocol. The within-pair $\Delta$ per seed,

$$\Delta_s = M_s^\text{FedProx} - M_s^\text{FedAvg},$$

is computed for each condition $\in \{C0, C1, C2\}$. We then test two
hypotheses by paired Wilcoxon signed-rank:

**H1 (within-condition).** For each condition $c$, is $\Delta_s^c$
significantly different from zero? This is the same test reported for the
headline; positive significant result $\Rightarrow$ FedProx beats FedAvg
under that level of system heterogeneity.

**H2 (between-condition).** Does system heterogeneity amplify the FedProx
advantage compared to the no-system-het baseline? This is tested by paired
Wilcoxon on $\Delta_s^c - \Delta_s^{C0}$ for $c \in \{C1, C2\}$. A
positive $p < 0.05$ result supports the canonical FedProx claim that the
proximal regulariser is most beneficial under system heterogeneity.

In addition, we report the **straggler-tolerance ratio**

$$\rho_\text{algo}^c = \frac{M_\text{algo}^c}{M_\text{algo}^{C0}},$$

quantifying how much each algorithm's performance degrades relative to
its no-system-het baseline. The prediction is $\rho_\text{FedProx}^c > \rho_\text{FedAvg}^c$
for both system-het conditions.

## Compute cost and reproducibility

40 SLURM jobs (2 conditions × 10 paired seeds × 2 algorithms) at
approximately 1 GPU-hour each on a single A100 (Cambridge CSD3, Ampere
partition), totalling $\sim$40 GPU-hours. The full submission is
reproduced by

```bash
bash mnist_dermnist/scripts/submit_system_het.sh
```

Outputs land in `mnist_dermnist/results/system_het_fixed/` and
`mnist_dermnist/results/system_het_random/` with filenames tagged
`sh-fixed_stragglers` and `sh-random_stragglers` respectively, so they
cannot be confused with the statistical-heterogeneity sweep outputs.

## Threats to validity

**Within-partition confound.** Holding the partition constant ensures the
system-heterogeneity contribution is isolated, but means we cannot say
whether system heterogeneity has the same effect under different statistical
heterogeneities. A 2×2 factorial design (system het × IID/non-IID) would
address this but is identified as future work.

**Straggler-fraction choice.** We test only 50% random stragglers
(matching Li et al. 2020's setup) and 2 of 7 fixed stragglers. Other
fractions (25%, 75%, 90%) would map the FedProx advantage as a function
of straggler severity but were not run due to GPU-budget constraints.

**$E_\text{straggler}$ choice.** Fixed-straggler runs use $E_\text{straggler} = 5$;
random-straggler runs draw $E_i \sim \text{Uniform}[1, 19]$. Different
choices would change the magnitude of the effect; the qualitative
prediction (FedProx > FedAvg under stragglers) should not depend on these
choices but the quantitative magnitudes will.

**Single $\mu$.** Like the headline experiment, $\mu = 0.01$ is fixed. A
larger $\mu$ might help FedProx more under severe system heterogeneity by
anchoring stragglers more tightly; this is identified as future work.
